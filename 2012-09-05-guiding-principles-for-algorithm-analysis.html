Guiding Principles for Algorithm Analysis

Wed 05 September 2012
By cgopalan

In text.

tags: algorithms

Three guiding principles:

1. Worst case analysis: Requires no assumptions on where input comes from. Also no domain knowledge needed.

Theres also average case analysis/benchmarks. This requires domain knowledge

  1. Not much attention to constant factors, lower-order terms.

  1. Asymptotic analysis - focus on large input sizes.

What is a "fast" algorithm?

One whose worst-case running time grows slowly with input size

Holy grail: achieve linear running time (or close to it)
